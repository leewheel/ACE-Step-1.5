# 🚀 立即开始使用 ACE-Step 1.5

## ⚡ 3 步启动（最简单）

### 第 1 步：双击启动

```
双击运行：start_gradio_ui.bat
```

### 第 2 步：等待启动

首次启动会自动：
- 读取 `.env` 配置（中文界面、端口 7861）
- 下载模型（如果需要，10-30 分钟）
- 初始化服务

### 第 3 步：打开浏览器

```
http://localhost:7861
```

**就这么简单！** 🎉

## 🎵 生成第一首歌（30 秒）

### 简单模式（推荐）

1. 在界面选择"生成模式" → **简单**
2. 输入描述：
   ```
   一首温柔的中文流行情歌，钢琴伴奏，抒情动人
   ```
3. 选择"人声语言" → **zh (Chinese)**
4. 点击"创建样本"
5. 点击"生成音乐"
6. 等待 10-15 秒，完成！

---

## 🎤 尝试 Cover 翻唱

### 将英文歌改成中文版

1. 选择"生成模式" → **Remix**
2. 上传英文歌曲
3. 音乐描述：
   ```
   中文流行翻唱版本，保留原曲旋律，情感丰富的人声
   ```
4. 填入中文歌词
5. 人声语言：**zh (Chinese)**
6. Remix 强度：**0.7**
7. 翻唱强度：**0.2**
8. 点击"生成音乐"

## 🔧 如果遇到问题

### 问题 1：端口被占用（502 错误）

**快速解决**：

编辑 `.env` 文件，改为其他端口：
```env
PORT=8080
```

然后重新运行 `start_gradio_ui.bat`

### 问题 2：界面是英文

**检查**：

打开 `.env` 文件，确认有：
```env
LANGUAGE=zh
```

如果没有，添加这一行，保存，重新启动。

### 问题 3：启动很慢

**正常现象**：

首次启动需要下载模型（约 10-30 分钟）。
后续启动只需 30-60 秒。

## 💡 快速提示

### 生成高质量音乐

- 描述要详细（流派、乐器、情绪、人声风格）
- 使用 SFT 模型（在服务配置中选择）
- 批量生成 2-4 个版本，选择最好的

### Cover 翻唱技巧

- 新歌词节奏与原曲相似
- Remix 强度：0.6-0.7（保留结构）
- 翻唱强度：0.15-0.25（保留旋律）
- 使用 SFT 模型效果最好

### 多语言控制

**中文**：
- 人声语言选 `zh`
- 描述加 `Chinese Mandarin pop`

**粤语**：
- 人声语言选 `yue`
- 描述加 `Hong Kong Cantopop`

**英语**：
- 人声语言选 `en`
- 描述加 `English pop`

## 🎯 配置说明

你的 `.env` 文件已优化为：

```env
LANGUAGE=zh                              # 中文界面
PORT=7861                                # 避免端口冲突
ACESTEP_CONFIG_PATH=acestep-v15-turbo   # Turbo 模型（快速）
ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-0.6B  # 0.6B LM（适合 8GB）
ACESTEP_LM_BACKEND=pt                    # PyTorch 后端（避免 Triton）
ACESTEP_INIT_LLM=true                    # 启用 LM（Think 模式）
```

**预期性能**（RTX 4060 8GB）：
- 生成时间：10-15 秒/首
- 显存占用：6-7 GB
- 支持功能：全部（包括 Cover/Remix）
