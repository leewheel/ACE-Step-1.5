## 🔧 解决方案

### 方案 1：更改端口（推荐）

创建 `.env` 文件并设置不同的端口：

```bash
# 复制示例配置
cp .env.example .env
```

编辑 `.env` 文件，添加或修改以下内容：

```env
# 界面语言设置为中文
LANGUAGE=zh

# 更改端口（避免冲突）
PORT=7861

# 或者尝试其他端口
# PORT=8080
# PORT=8888
```

然后重新启动：
```bash
uv run acestep
```

### 方案 2：检查端口占用

**检查端口是否被占用**：
```powershell
netstat -ano | findstr :7860
```

如果端口被占用，你会看到类似输出：
```
TCP    127.0.0.1:7860    0.0.0.0:0    LISTENING    12345
```

**关闭占用端口的进程**：
```powershell
# 使用上面输出的 PID（例如 12345）
taskkill /PID 12345 /F
```

### 方案 3：使用命令行参数指定端口

直接在启动时指定端口：
```bash
uv run acestep --port 7861
```

或者：
```bash
uv run acestep --server-name 0.0.0.0 --port 8080
```

### 方案 4：禁用 LM 快速启动（如果问题持续）

如果上述方法都不行，可以先禁用 LM 快速测试：

在 `.env` 文件中添加：
```env
LANGUAGE=zh
PORT=7861
ACESTEP_INIT_LLM=false
```

这样可以更快启动，先确保基础功能正常。

## 📝 关于 Triton 警告的说明

### 什么是 Triton？
Triton 是 NVIDIA 的 GPU 加速库，用于优化 vLLM 后端的性能。

### 为什么会出现这个警告？
在 Windows 上，Triton 的支持不完整，这是已知问题。

### 影响是什么？
- ✅ **功能完全正常**：系统自动回退到 PyTorch 后端
- ⚠️ **性能稍慢**：vLLM 后端比 PyTorch 后端快约 20-30%
- 💡 **推荐方案**：使用 PyTorch 后端（已自动启用）

### 如何明确使用 PyTorch 后端？

在 `.env` 文件中设置：
```env
LANGUAGE=zh
PORT=7861
ACESTEP_LM_BACKEND=pt
```

这样会直接使用 PyTorch 后端，避免 Triton 警告。

## 🚀 完整的启动步骤（推荐）

### 步骤 1：创建配置文件

```bash
cp .env.example .env
```

### 步骤 2：编辑 `.env` 文件

使用文本编辑器打开 `.env`，设置以下内容：

```env
# ==================== 界面语言 ====================
LANGUAGE=zh

# ==================== 服务器设置 ====================
# 更改端口避免冲突
PORT=7861

# 如果需要从其他设备访问，设置为 0.0.0.0
# SERVER_NAME=0.0.0.0

# ==================== 模型设置 ====================
# DiT 模型（推荐 turbo 快速生成）
ACESTEP_CONFIG_PATH=acestep-v15-turbo

# LM 模型（根据你的 8GB GPU，使用 0.6B）
ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-0.6B

# LM 后端（使用 pt 避免 Triton 警告）
ACESTEP_LM_BACKEND=pt

# ==================== LLM 初始化 ====================
# 启用 LM（Think 模式）
ACESTEP_INIT_LLM=true

# ==================== 设备设置 ====================
ACESTEP_DEVICE=auto
```

### 步骤 3：启动服务

```bash
uv run acestep
```

### 步骤 4：访问界面

打开浏览器：http://localhost:7861

（注意端口改为 7861）

## 🐛 其他常见问题

### Q1：界面显示英文而不是中文

**原因**：日志显示 `Creating Gradio interface with language: en`

**解决方案**：
1. 确保 `.env` 文件中设置了 `LANGUAGE=zh`
2. 如果使用启动脚本，检查脚本是否覆盖了语言设置
3. 重新启动服务

### Q2：GPU 显存不足

**症状**：
```
CUDA out of memory
```

**解决方案**：
在 `.env` 中添加：
```env
# 禁用 LM，只使用 DiT
ACESTEP_INIT_LLM=false

# 或者使用更小的 LM 模型
ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-0.6B
ACESTEP_LM_BACKEND=pt
```

### Q3：模型下载失败

**症状**：
```
Failed to download model
```

**解决方案**：
1. 检查网络连接
2. 使用国内镜像（ModelScope）：
   ```env
   ACESTEP_DOWNLOAD_SOURCE=modelscope
   ```
3. 手动下载模型到 `checkpoints/` 目录

### Q4：生成速度很慢

**原因**：
- 使用 CPU 而非 GPU
- 使用 Base 模型而非 Turbo
- LM 后端使用 PyTorch 而非 vLLM

**解决方案**：
1. 确认 GPU 可用：
   ```bash
   uv run python -c "import torch; print(torch.cuda.is_available())"
   ```
2. 使用 Turbo 模型：
   ```env
   ACESTEP_CONFIG_PATH=acestep-v15-turbo
   ```
3. 减少推理步数（Turbo 模型使用 4-8 步）

### Q5：启动时卡住不动

**可能原因**：
- 首次启动正在下载模型
- 网络连接问题

**解决方案**：
1. 耐心等待（首次下载可能需要 10-30 分钟）
2. 检查网络连接
3. 查看日志输出，确认下载进度
