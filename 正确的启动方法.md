# ✅ 正确的启动方法

## 🎯 问题分析

你遇到的问题：
1. ❌ 界面显示英文（`Creating Gradio interface with language: en`）
2. ❌ 端口仍然是 7860（`Launching server on 127.0.0.1:7860`）
3. ❌ 启动脚本中文字符乱码

## 💡 解决方案

**不要使用 `启动-中文界面.bat`**，而是使用原始的 `start_gradio_ui.bat`！

原始脚本已经内置了读取 `.env` 文件的功能，会正确应用你的配置。

---

## 🚀 正确的启动步骤

### 方式 1：使用原始启动脚本（推荐）

```bash
# 双击运行
start_gradio_ui.bat
```

或者在命令行中：
```bash
.\start_gradio_ui.bat
```

### 方式 2：使用命令行

```bash
uv run acestep --language zh --port 7861
```

### 方式 3：使用 uv run 直接启动

```bash
uv run acestep
```

**注意**：方式 3 不会自动读取 `.env` 文件，需要手动指定参数。

---

## 📋 验证配置

启动后，你应该看到：

```
[Config] Loading configuration from .env file...
[Config] Configuration loaded from .env
...
Creating Gradio interface with language: zh...
Launching server on 127.0.0.1:7861...
* Running on local URL:  http://127.0.0.1:7861
```

关键点：
- ✅ `language: zh`（不是 en）
- ✅ `7861`（不是 7860）

## 🔧 如果仍然有问题

### 问题 1：端口 7861 仍然被占用

**解决方案**：在 `.env` 中改为其他端口

```env
PORT=8080
```

或者：
```env
PORT=8888
```

### 问题 2：界面仍然是英文

**检查 `.env` 文件**：
```bash
# 打开 .env 文件，确认有这一行：
LANGUAGE=zh
```

**手动指定语言**：
```bash
start_gradio_ui.bat
# 或
uv run acestep --language zh --port 7861
```

### 问题 3：Gradio 启动失败（502 错误）

这是端口冲突问题。解决方法：

**方法 1：关闭占用端口的进程**
```powershell
# 查找占用端口的进程
netstat -ano | findstr :7860

# 关闭进程（替换 PID 为实际的进程 ID）
taskkill /PID <PID> /F
```

**方法 2：使用不同的端口**

在 `.env` 中修改：
```env
PORT=8080
```

## 📝 `.env` 文件完整配置

确保你的 `.env` 文件包含以下内容：

```env
# 界面语言（中文）
LANGUAGE=zh

# 服务器端口（避免冲突）
PORT=7861

# 服务器地址
SERVER_NAME=127.0.0.1

# DiT 模型
ACESTEP_CONFIG_PATH=acestep-v15-turbo

# LM 模型
ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-0.6B

# LM 后端（PyTorch，避免 Triton 问题）
ACESTEP_LM_BACKEND=pt

# 启用 LM
ACESTEP_INIT_LLM=true

# 设备
ACESTEP_DEVICE=auto

# 批量大小
ACESTEP_BATCH_SIZE=1
```

## ✅ 完整的启动流程

### 第 1 步：确认配置文件

```bash
# 检查 .env 文件是否存在
dir .env

# 如果不存在，复制示例文件
copy .env.example .env
```

### 第 2 步：编辑配置（如果需要）

使用记事本打开 `.env`，确认：
- `LANGUAGE=zh`
- `PORT=7861`（或其他未被占用的端口）

### 第 3 步：启动服务

```bash
# 使用原始启动脚本
start_gradio_ui.bat
```

### 第 4 步：访问界面

打开浏览器：
```
http://localhost:7861
```

（如果你改了端口，使用对应的端口号）

---

## 🎯 关键要点

1. **使用 `start_gradio_ui.bat`**，不要使用 `启动-中文界面.bat`
2. **`.env` 文件必须存在**，并且配置正确
3. **端口不能被占用**，如果 7861 被占用，改用其他端口
4. **首次启动需要下载模型**，可能需要 10-30 分钟
